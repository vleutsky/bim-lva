# Локальный ИИ в BIM.LVA Composer

Полная инструкция: что скачать, установить и как пользоваться ассистентом анализа модели (чат, коллизии→BCF, фильтры, действия UI).

Данные модели **не уходят в облако**: запросы идут на `127.0.0.1` (Ollama на вашем ПК).

---

## 1. Что получится

В меню **Анализ → ИИ (локально)** откроется панель чата.

Ассистент умеет (поверх ваших инструментов, не вместо них):

1. Отвечать на вопросы по сводке IFC (классы, GlobalId, выделение)
2. Помогать формулировать выводы по коллизиям / замечаниям
3. Предлагать черновик BCF (заголовок + комментарий)
4. Кратко резюмировать контекст сравнения / выделения
5. Пояснять ведомость «на пальцах» по текущему выделению
6. Вызывать действия UI: выделить по GlobalId, isolate, показать всё, вписать в кадр, черновик BCF

> Важно: ИИ **не смотрит** 3D-mesh. Он работает с JSON-индексом, который Composer собирает из уже загруженной модели.

## 2. Что скачать и установить

### A. Node.js (LTS)

1. Откройте https://nodejs.org  
2. Скачайте **LTS** (не Current)  
3. Установите с настройками по умолчанию  
4. Проверка в терминале:

```bash
node -v
```

Должно показать v20/v22/….

### B. Ollama (локальный LLM runtime)

1. Откройте https://ollama.com/download  
2. Установите для Windows / macOS / Linux  
3. Запустите Ollama (в Windows обычно появляется в трее)  
4. Проверка:

```bash
ollama --version
```

### C. Модель (один раз скачать веса)

Рекомендации:

| ПК | Команда | Комментарий |
|---|---|---|
| 8–16 ГБ RAM | `ollama pull llama3.2` | хороший старт |
| 16+ ГБ / есть GPU | `ollama pull qwen2.5:14b` | лучше рассуждает |
| слабый ноутбук | `ollama pull llama3.2:1b` | быстрее, проще |

Пример:

```bash
ollama pull llama3.2
```

Проверка списка:

```bash
ollama list
```

Тест чата в терминале (необязательно):

```bash
ollama run llama3.2
```

Выход: `/bye`.

### D. Репозиторий BIM.LVA

Нужна локальная копия проекта (не только GitHub Pages):

- клон GitHub, или  
- скачать ZIP репозитория и распаковать.

В корне должны быть файлы:

- `bim-lva-composer-ifc.html`
- `ai-client.js`
- `ai-bridge/server.mjs`
- `AI-LOCAL.md` (этот файл)

## 3. Запуск bridge (обязательно)

Bridge делает две вещи:

1. Раздаёт Composer по `http://127.0.0.1:3847` (иначе с GitHub Pages браузер **блокирует** обращение к localhost)
2. Проксирует чат в Ollama

### Windows

Дважды кликните:

```text
ai-bridge\start.bat
```

или в PowerShell из корня репозитория:

```powershell
node ai-bridge\server.mjs
```

### macOS / Linux

```bash
chmod +x ai-bridge/start.sh
./ai-bridge/start.sh
```

или:

```bash
node ai-bridge/server.mjs
```

В консоли должно появиться:

```text
UI:     http://127.0.0.1:3847/bim-lva-composer-ifc.html
Health: http://127.0.0.1:3847/ai/health
```

**Окно bridge не закрывайте**, пока пользуетесь ИИ.

Проверка в браузере: откройте http://127.0.0.1:3847/ai/health  
Ожидается `"ok": true`, `"ollama": true` и список моделей.

---

## 4. Открыть Composer правильно

Открывайте **только через bridge**:

http://127.0.0.1:3847/bim-lva-composer-ifc.html

Не используйте `https://…github.io/...` для ИИ-режима: mixed content заблокирует localhost.

В HUD build должен быть с тегом вроде `local-ai-v1` (после деплоя/обновления файлов).

---

## 5. Как пользоваться в интерфейсе

1. Загрузите IFC (Локально / Я.Диск)  
2. **Анализ → ИИ (локально)**  
3. Нажмите **Проверить связь** — статус станет зелёным  
4. Выберите модель (если пусто — `llama3.2`)  
5. Примеры запросов:

- `Сколько элементов в индексе и какие IFC-классы чаще встречаются?`
- `Выдели стены (IfcWall) если они есть в контексте`
- `Сформулируй черновик BCF по текущему выделению`
- `Что делать дальше с коллизией / замечанием?` (если есть контекст)

Если модель вернула `actions`, Composer выполнит их (выделение, isolate, draft BCF и т.д.).

## 6. Архитектура (как это устроено)

```text
[Composer в браузере]
   │  JSON-контекст модели + сообщение пользователя
   ▼
[ai-bridge :3847]  ──статика──► bim-lva-composer-ifc.html
   │
   ▼
[Ollama :11434]  (llama3.2 / qwen / …)
   │
   ▼
текст ответа + JSON actions
   │
   ▼
Composer исполняет tools (select / isolate / BCF draft / …)
```

Файлы:

| Путь | Назначение |
|---|---|
| `ai-bridge/server.mjs` | HTTP bridge + proxy Ollama |
| `ai-bridge/start.bat` / `start.sh` | быстрый запуск |
| `ai-client.js` | клиент в браузере |
| `AI-LOCAL.md` | эта инструкция |

## 7. Переменные окружения (опционально)

```bash
# порт bridge
set BIMLVA_AI_PORT=3847

# адрес Ollama (если не стандартный)
set OLLAMA_HOST=http://127.0.0.1:11434

# модель по умолчанию
set BIMLVA_AI_MODEL=llama3.2
```

На macOS/Linux: `export VAR=value`.

## 8. Типичные проблемы

### `Bridge недоступен`
- не запущен `node ai-bridge/server.mjs`
- открыли GitHub Pages вместо `http://127.0.0.1:3847/...`
- порт 3847 занят → смените `BIMLVA_AI_PORT`

### `ollama: false` в `/ai/health`
- Ollama не запущена
- модель не скачана: `ollama pull llama3.2`
- Ollama на другом порту → `OLLAMA_HOST`

### Ответы «глупые» / пустые actions
- дайте более конкретный запрос и выделите элементы на сцене
- смените модель на более крупную (`qwen2.5:14b`)
- ИИ не знает то, чего нет в JSON-контексте (ограниченный индекс)

### Антивирус / корпоративная сеть
- разрешите Node и Ollama localhost
- не нужен интернет после скачивания модели (кроме первого pull)

### GitHub Pages и ИИ одновременно
- Pages — для обычной работы  
- ИИ-режим — только через local bridge  
(позднее можно упаковать в Tauri/Electron, если нужен один ярлык)

---

## 9. Безопасность и границы

- Ключи облачных LLM **не нужны**
- Запросы по умолчанию не покидают ПК
- Не доверяйте ИИ юр.экспертизе «вслепую»: нормы и коллизии подтверждайте своими инструментами Composer
- Не вставляйте в чат секреты/пароли

---

## 10. Быстрый чеклист «с нуля»

1. Установить Node.js LTS  
2. Установить Ollama  
3. `ollama pull llama3.2`  
4. Открыть папку репозитория  
5. `node ai-bridge/server.mjs`  
6. Открыть http://127.0.0.1:3847/bim-lva-composer-ifc.html  
7. Загрузить IFC  
8. **Анализ → ИИ (локально)** → Проверить связь → спросить
