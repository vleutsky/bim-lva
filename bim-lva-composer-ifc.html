Revit / Tekla / Plant 3D

Нативные .rvt, модели Tekla и Autodesk Plant 3D — проприетарные форматы. В браузере их нельзя разобрать без SDK вендора.

Сейчас: экспортируйте в IFC 2x3 / IFC4 и откройте здесь.
  • Revit → File → Export → IFC
  • Tekla → Export → IFC
  • Plant 3D → Export to IFC / Navisworks → IFC

Заготовка на будущее: облачный конвейер Autodesk APS / Trimble Connect (загрузка → перевод в IFC/SVF → открытие в Composer). Серверная часть пока не подключена.

Яндекс.Диск — облачная библиотека моделей

Как настроить (один раз):
  1. На disk.yandex.ru создайте папку, например BIM.LVA Models.
  2. Загрузите туда IFC / FBX / DXF / GLB и георастры (GeoTIFF, ECW, JPEG/PNG + world-файл .jgw/.pgw).
  3. ПКМ по папке → Поделиться → скопируйте публичную ссылку вида https://disk.yandex.ru/d/XXXX.
  4. Вставьте ссылку ниже и нажмите Открыть папку — выберите файл и загрузите по одному (так экономится память).
OAuth-токен не нужен: Composer работает через публичный API Яндекс.Диска.

ECW / GeoTIFF — растр по координатам на рельеф

Формат ECW хранит геопривязку внутри файла — растр автоматически встанет в координаты сцены и ляжет на 3D-рельеф. Также поддерживаются GeoTIFF и JPEG/PNG с world-файлом (.jgw / .pgw / .tfw / .eww). Если ECW не откроется в браузере, экспортируйте тот же снимок в GeoTIFF с геопривязкой — координаты сохранятся.
